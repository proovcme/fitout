# FITOUT

Игровой вертикальный срез про технического заказчика офисных ремонтов. Игрок выбирает заказ на глобальной карте, торгуется за бюджет и сроки, собирает команду и подрядчиков, а затем управляет объектом непрямо: расставляет утренние приоритеты, пишет в чат и почту и вечером сдаёт абсурдно подробный отчёт. Бригады автономно работают на процедурном 3D-объекте Three.js.

Версия `0.0.7` добавляет постоянную кнопку меню в верхнюю панель, усиливает читаемость ходьбы и рабочих действий юнитов и заменяет условные коробки во дворе на узнаваемый грузовик и сервисный фургон с кабинами, стёклами, фарами, колёсами и отдельными маршрутами движения.

Версия `0.0.6` добавила живой фон объекта: контекстные микроэпизоды между крупными событиями, производственные планёрки, замеры, доставки, проверки, споры и тесты инженерии. Они отображаются в 3D действиями людей, техникой, светом, инструментами и короткой плашкой состояния, но не требуют постоянного выбора игрока. Перегородки физически проходят стадию металлического каркаса до обшивки, камера мягко следует за активностью, а свет меняется в течение смены.

Первая учебная миссия, связанная кампания из четырёх заказов, развитие собственного офиса и физическая визуализация стадий остаются основой среза: доставка материалов, перегородки, инженерия, отделка, мебель, уборка и сдача. Юниты носят материалы, красят, сверлят, спорят, курят и простаивают; решения игрока сопровождаются визуальным импульсом, реакцией камеры и процедурным звуком.

В сборке есть 50 случайных событий и отдельный банк из 24 производственных ситуаций. Ситуация появляется знаком `!` над конкретным персонажем; игрок отвечает на вопрос подрядчика, а нанятый профильный специалист может решить часть вопросов самостоятельно. За день случается не больше двух крупных событий, иногда ни одного; один девятичасовой рабочий день длится пять реальных минут на скорости `1×`, а окна решений, чат и почта останавливают время.

Общий график генерируется по зависимостям и редактируется игроком; утренняя планёрка получает задачи из него, а завершённые работы больше не повторяются. Проект, исполнительная документация, зарплаты, материалы, мобилизация, аванс и этапные платежи участвуют в общей симуляции. Между объектами сохраняется экономика организации: прибыль и убытки, кредиты с процентами, долг, репутация и история проектов.

## Локальный запуск

```bash
npm install
npm run dev
```

Открыть `http://127.0.0.1:4173/fg/`.

## Проверки

```bash
npm test
npm run build
npm audit --audit-level=moderate
```

Статическая production-сборка создаётся в `dist/`. Профили игроков, сохранения и история миссий обслуживаются небольшим API из `server/server.py` (production) или `server/server.mjs` (локальная Node.js-версия) по префиксу `/fg-api`.

## Production

Публичная версия: `https://les.ovc.me/fg/`.

- Клиент разворачивается атомарно из версионированных каталогов `/var/www/fitout/releases/`.
- Текущий клиент подключён симлинком `/var/www/fitout/current`.
- API работает только на loopback-порту `4188` и публикуется Caddy по `/fg-api/`.
- Постоянные данные находятся в `/var/lib/fitout`, служба — `fitout-save.service`.
